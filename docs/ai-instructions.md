# Правила разработки компонентов

## 1. Разделение логики и UI

### Компоненты должны быть разделены на:
- **UI-компоненты** (`src/components/`):
  - Только отображение и стилизация
  - Принимают пропсы и колбэки
  - Не содержат бизнес-логику
  - Пример: `LogoutButton`, `UserAvatar`

- **Логические компоненты** (`src/app/`):
  - Управление состоянием
  - Работа с API
  - Бизнес-логика
  - Пример: `DashboardPage`, `AuthProvider`

## 2. Структура компонентов

### Каждый компонент должен:
1. Иметь четкую зону ответственности
2. Использовать строгую типизацию
3. Следовать принципу единой ответственности
4. Быть переиспользуемым

### Пример правильной структуры:
```typescript
// UI-компонент
interface ButtonProps {
  onClick: () => void;
  children: ReactNode;
}

export function Button({ onClick, children }: ButtonProps) {
  return <button onClick={onClick}>{children}</button>;
}

// Логический компонент
export function DashboardPage() {
  const handleLogout = () => {
    // Бизнес-логика
  };

  return <Button onClick={handleLogout}>Выйти</Button>;
}
```

## 3. Стилизация

### Использовать Tailwind CSS:
- Компоненты должны быть адаптивными
- Использовать `flex`, `grid`, `max-w-screen-xl`
- Добавлять анимации через `transition` и `animate`
- Следовать дизайн-системе проекта

## 4. Типизация

### Строгая типизация:
- Все пропсы должны быть типизированы
- Использовать типы из `@/types/`
- Избегать `any`
- Использовать `Record` для маппингов

## 5. Обработка ошибок

### Каждый компонент должен:
- Обрабатывать ошибки
- Показывать состояния загрузки
- Иметь фолбэки для отсутствующих данных

## 6. Тестирование

### Компоненты должны быть:
- Легко тестируемыми
- Иметь минимальные зависимости
- Использовать моки для внешних сервисов
